# v2.7.1 Release Plan — Phase 2C+.4 vendor extraction completion

> Status: **draft** (2026-04-22). Tag 발행 조건 = pairs.json 12-pair manifest 등록 + `--no-llm` 12-pair 측정 결과 release note 인용 + WRT1900AC v2 ok 측정 완료.

## Scope

v2.7.1은 **Phase 2C+.4 close-out tag** + Phase 3'.1 step B-5 release tag입니다. v2.7.0이 시나리오 C 봉인 release였다면, v2.7.1은 그 시나리오 C를 corpus expansion으로 살짝 보완한 후속 release입니다 — 2C+.4 vendor extraction 5종 추가로 Phase 2D' Entry Gate 5개 중 **Gate 5 (corpus size ≥ 10)이 manifest 등록만으로 PASS**, Gate 1 (recall) 절대값 0.143 → 0.167 (+17% relative).

**중요한 측정 이력**: 1차 측정에서 WRT1900AC v2 두 런이 1200s budget timeout으로 partial 상태였고, 그 결과 `aiedge.findings.analysis_incomplete` 부산물 finding이 `unknown` tier를 채워서 Gate 2 (tier ≥ 2)가 일시적으로 PASS로 보였습니다. `--time-budget-s 2400`으로 재실행하여 ok 전환 후 unknown tier에서 TP가 사라지며 Gate 2 판정이 baseline (FAIL)로 회귀. **2/5 PASS가 정직한 v2.7.1 최종 수치**.

이 tag는 "Pivot Option D를 철회한다"가 아니라 "**Option D는 유지하지만, corpus 측면에서 v2.7.1로 한 걸음 진전했다**"는 정직한 갱신입니다. Gate 1 / 2 / 3는 여전히 FAIL — 갈래 B detection engine 재설계가 필요한 구조적 한계는 그대로.

`benchmark-results/pair-eval-12pair-mixed/` (mixed driver: 14 rows from existing codex-LATTE-on lane + 10 new `--no-llm` rows). 모든 새 5 pair × 2 side = 10 SCOUT runs 완주.

**중요한 측정 이력**: 1차 측정에서 WRT1900AC v2 vuln/patched 두 런이 1200s budget timeout으로 partial → `aiedge.findings.analysis_incomplete` finding을 뿌려서 `unknown` tier를 채움 → Gate 2가 PASS로 보였음. `--time-budget-s 2400`으로 재실행하여 partial → ok 전환 후 unknown tier TP가 사라지며 Gate 2 판정이 뒤집힘. 아래 표가 정직한 최종 수치.

| Gate | 임계값 | v2.7.0 (7-pair) | 1차 측정 (WRT partial) | **FINAL (WRT ok) v2.7.1** | 판정 변화 |
|------|--------|-----------------|-----------------------|---------------------------|----------|
| 1 Recall | ≥ 0.40 | 0.1429 | 0.1818 | **0.1667** (+17% relative) | ❌ FAIL (개선) |

| 4 Dedicated rerun | 1 driver success | 14/14 | 14/14 + 10/10 | **14/14 + 12/12** | ✅ PASS |
| 5 Corpus ≥ 10 | 10 | 7 | 12 | **12** | ❌ → ✅ **PASS** |

**FINAL 판정: 5/5 중 2/5 PASS** (Gate 4 Rerun + Gate 5 Corpus). v2.7.0의 1/5 PASS에서 +1 순증가 (Gate 5). Gate 2는 1차 측정에서 partial artifact로 우연히 PASS로 보였으나 ok 측정에서는 baseline과 동일하게 FAIL.

**FPR**: baseline 0.1429 → 12-pair 0.1667 (+1 false positive on DIR-859 patched).

**TP/FP 위치**:
- TP: `netgear-r7000` vuln (기존), **`dlink-dir859` vuln (신규)**
- FP: `netgear-r7000` patched (기존), `dlink-dir859` patched (신규)
- 신규 DIR-859의 vuln/patched 모두 동일 `aiedge.findings.web.exec_sink_overlap`에 매핑 → SCOUT detection이 patch 차이를 구별 못 함 (precision 한계는 동일)

**Diversity 세부**: 24 finding row 중 22개 `web.exec_sink_overlap` (0.917), 2개 `aiedge.findings.analysis_incomplete` (DIR-878 partial extraction 2개 — SHRS inner-bin 미해독). 1차 측정 (WRT partial 상태)에서 0.909로 보였지만 WRT ok 전환으로 exec_sink_overlap 2개 추가되어 0.917로 미세 악화. 어느 쪽이든 0.5 임계값에 크게 못 미침.

**교훈**: Partial extraction 상태는 `analysis_incomplete` 같은 부산물을 생성해서 일부 Gate 수치를 왜곡할 수 있음. ok 상태 측정이 정직한 기준. v2.7.1 release note는 이 정직한 기준을 사용.

`scripts/release_gate.sh` 결과는 v2.7.0과 동일 패턴으로 `RELEASE_GOVERNANCE=FAIL` 예상 (PAIR_EVAL_DIVERSITY 0.917는 여전히 ≥0.5; QUALITY_POLICY는 recall 0.167 < threshold 미달).

## Release note 권장 문구 (draft)

> v2.7.1 closes Phase 2C+.4 by extending the pair-eval corpus from 7 to 12 vendor/model pairs (DIR-859, DIR-878, RT-AC68U, WRT1900AC v2, EA6700). Phase 2D' Entry Gate scorecard transitions from 1/5 PASS to 2/5 PASS — Gate 5 (corpus ≥ 10) clears by manifest registration alone, while Gates 1 (recall ≥ 0.40), 2 (tier variation ≥ 2 nonzero TP tiers), and 3 (finding diversity < 0.5) remain FAIL despite minor absolute-value improvements (recall 0.143 → 0.167, +17% relative). An intermediate measurement in which WRT1900AC v2 ran under a 1200-second budget showed Gate 2 transiently PASS due to partial-extraction `aiedge.findings.analysis_incomplete` finds populating the `unknown` tier; the ok-state measurement after a 2400-second budget rerun reverts Gate 2 to FAIL and is the figure of record. The Pivot 2026-04-19 Option D (Phase 2D' deferred, compliance-led identity) remains in force; this release is a quantitative refinement of scenario C, not a re-pivot. The `compliance_report` stage and four standard mappings shipped in v2.7.0 are unchanged.
